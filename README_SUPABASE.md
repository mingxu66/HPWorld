# Supabase 数据库存档设置

这版支持 Render 后端 + Supabase 免费数据库。玩家存档会写入 Supabase 的 `player_saves` 表，不再依赖 Render 免费服务器的临时文件系统。

## 1. 在 Supabase 运行 SQL

打开 Supabase → SQL Editor → New query，运行：

```sql
create table if not exists public.player_saves (
  id bigint generated by default as identity primary key,
  user_id text not null default 'default_user',
  slot_id text not null,
  save_name text,
  save_data jsonb not null,
  created_at timestamptz default now(),
  updated_at timestamptz default now(),
  unique(user_id, slot_id)
);

alter table public.player_saves enable row level security;

create policy if not exists "service role can manage player saves"
on public.player_saves
for all
using (true)
with check (true);
```

## 2. 在 Render 填环境变量

Render → hpWorld → Environment → Add variable：

```text
SUPABASE_URL=https://你的项目id.supabase.co
SUPABASE_SERVICE_ROLE_KEY=sb_secret_你的secret key
```

填完后点：Save, rebuild, and deploy。

## 3. 检查是否启用数据库

打开：

```text
https://你的render地址/api/health
```

如果看到：

```json
"storage":"supabase"
```

就说明已经启用 Supabase 存档。

如果看到：

```json
"storage":"local-file-fallback"
```

说明 Render 环境变量还没填好，或者还没重新部署。

## 4. 说明

- GitHub Pages 不能运行后端，Supabase 存档只在 Render 网站生效。
- 不要把 `SUPABASE_SERVICE_ROLE_KEY` 写进 GitHub 文件。
- 如果重新部署 Render，Supabase 里的存档不会丢。
